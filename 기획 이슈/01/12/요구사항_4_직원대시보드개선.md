

## 🎯 목표

### 사용자 스토리
직원으로서, 오늘 할 일과 마감일을 한눈에 파악하고 싶습니다.

### 핵심 목표
- 대시보드를 Todo 리스트 형태로 변경
- 상태별 맞춤 정보 표시
- 빠른 액션 커스터마이징

---

## 📚 용어 정의

| 용어 | 정의 |
| --- | --- |
| **Dashboard** | 직원 메인 대시보드 |
| **Quick_Action** | 빠른 액션 버튼 영역 |
| **Todo_List** | 할 일 목록 |

---

## 📋 상세 요구사항

### 4-1. 대시보드 구성 변경

#### 현재 상황
- 오늘 연재 일정 중심

#### 개선 사항
| 항목 | 설명 |
| --- | --- |
| 일정/스케줄표/Todo 리스트 | 형태 변경 |
| 오늘 할 일 목록 | 당일 작업 목록 표시 |
| 마감일 D-day 표시 | 마감일까지 남은 일수 |
| 진행 중인 작업 현황 | 현재 진행 중인 작업 |
| 다음 연재일까지 일정 | 연재일 기준 일정 표시 |

---

### 4-2. 상태별 관리

| 상태 | 표시 내용 |
| --- | --- |
| 워케이션 중 | 원격 근무 현황, 체크인/체크아웃 상태 |
| 일반 근무 중 | 오늘의 작업 목록, 마감일 현황 |

---

### 4-3. 내용 관리 개선

| 항목 | 개선 사항 |
| --- | --- |
| 표시 중심 | 승인 대기 → 작업 현황, 마감일, 진행률 중심 |
| 빠른 액션 | 사용자 커스터마이징 가능 |
| 기능 추가 | 자주 사용하는 기능을 빠른 액션에 추가 |
| 순서 변경 | 드래그 앤 드롭으로 순서 변경 |

---

## 🎨 UI/UX 개선 포인트

- Todo 리스트 형태의 일정 관리
- 상태별 탭 구분 (워케이션 중 / 일반 근무 중)
- 빠른 액션 커스터마이징 UI
