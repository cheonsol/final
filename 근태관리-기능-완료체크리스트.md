# 📋 근태 관리 시스템 완료 체크리스트

## 📌 프로젝트 개요

본 문서는 **웹 작가 에이전시 HR SaaS 서비스**의 핵심 기능인 **근태 관리 시스템**에 대해 완료되어야 할 모든 기능을 체계적으로 정리한 문서입니다.

---

## 📌 1. 근태 유형 관리 기능

### 1-1. 근태 유형 정의 및 관리

#### 필수 근태 유형
- [ ] **반차** (오전반차 / 오후반차)
- [ ] **월차** (월 단위 휴가)
- [ ] **연차** (연 단위 휴가)
- [ ] **휴가** (연속 휴가)
- [ ] **워케이션** (원격 근무 형태의 휴양지 근무)

#### 근태 유형별 설정
- [ ] 각 근태 유형별 최대 신청 일수 제한 설정
- [ ] 근태 유형별 사전 신청 기간 설정 (예: 최소 3일 전)
- [ ] 근태 유형별 승인 단계 설정 (1단계/2단계 승인)
- [ ] 근태 유형별 필수 입력 항목 설정
- [ ] 근태 유형별 알림 설정 (승인자/신청자)

#### 완료 기준
- 모든 근태 유형이 시스템에 등록되어 관리 가능
- 관리자가 근태 유형별 설정을 유연하게 변경 가능
- 각 근태 유형별 정책이 명확히 적용됨

## 📌 2. 근태 신청 기능

### 2-1. 근태 신청 기본 기능

- [ ] **신청 폼 구성**
  - [ ] 근태 유형 선택 (반차/월차/연차/휴가/워케이션)
  - [ ] 시작일/종료일 선택 (날짜 선택기)
  - [ ] 반차인 경우 오전/오후 선택
  - [ ] 사유 입력 (필수/선택 설정 가능)
  - [ ] 첨부 파일 업로드 (의료진단서 등)
  - [ ] 대체 근무자 지정 (선택사항)

- [ ] **신청 유효성 검증**
  - [ ] 잔여 연차 일수 확인
  - [ ] 중복 신청 방지 (동일 기간)
  - [ ] 최소 신청 기간 확인 (사전 신청 정책)
  - [ ] 최대 신청 기간 확인 (연속 휴가 제한)
  - [ ] 휴일/공휴일 자동 제외 여부 설정

- [ ] **신청 상태 관리**
  - [ ] 상태: 대기 / 승인 / 반려 / 취소
  - [ ] 상태별 색상/아이콘 표시
  - [ ] 상태 변경 이력 추적

### 2-2. 워케이션 특화 기능

- [ ] **워케이션 신청 폼**
  - [ ] 근무 장소 입력 (주소/지명)
  - [ ] 지도 API 연동 (장소 선택)
  - [ ] 목적 선택 (업무 집중/환경 전환/기타)
  - [ ] 예상 업무 계획 입력
  - [ ] 연락처 정보 입력
  - [ ] 비상 연락망 정보
  - [ ] 원격 근무 환경 정보 (인터넷 속도, 작업 공간 등)

- [ ] **워케이션 원격 관리 기능** ⭐
  - [ ] **체크인/체크아웃 시스템**
    - [ ] 일일 출근 체크인 (위치 기반)
    - [ ] 일일 퇴근 체크아웃
    - [ ] 체크인/체크아웃 시간 기록
    - [ ] 위치 정보 자동 기록 (GPS)
    - [ ] 체크인 시 사진 업로드 (선택사항)
  
  - [ ] **일일 업무 보고**
    - [ ] 일일 업무 내용 입력
    - [ ] 업무 진행률 입력 (0-100%)
    - [ ] 완료한 작업 목록
    - [ ] 다음 날 계획 입력
    - [ ] 업무 보고 제출
  
  - [ ] **실시간 위치 관리** (선택적)
    - [ ] 실시간 위치 공유 (동의 시)
    - [ ] 지도에서 워케이션 중인 직원 위치 확인
    - [ ] 위치 이력 기록
    - [ ] 비정상 위치 알림 (신청 장소와 다를 경우)
  
  - [ ] **원격 근무 시간 관리**
    - [ ] 일일 근무 시간 기록
    - [ ] 주간 근무 시간 집계
    - [ ] 근무 시간 패턴 분석
    - [ ] 근무 시간 부족/초과 알림
  
  - [ ] **원격 근무 환경 모니터링**
    - [ ] 인터넷 연결 상태 체크
    - [ ] 작업 환경 사진 업로드
    - [ ] 업무 도구 확인 (노트북, 모니터 등)
    - [ ] 환경 이슈 보고

- [ ] **워케이션 관리 (관리자/승인자)**
  - [ ] 현재 워케이션 중인 직원 목록 조회
  - [ ] 워케이션 장소별 현황 조회
  - [ ] 워케이션 기간 중 업무 연계 확인
  - [ ] **원격 관리 대시보드** ⭐
    - [ ] 워케이션 중인 직원 실시간 현황
        - 어떻게 관리하는지 
        - 관리 유형 ~ 선택 하는게 있는건지 UI상 어떻게 표현할것인지 연구해서 
    - [ ] 체크인/체크아웃 현황 모니터링
    - [ ] 일일 업무 보고 확인
    - [ ] 위치 정보 확인 (동의 시)
    - [ ] 근무 시간 집계
    - [ ] 이상 징후 알림 (체크인 누락, 위치 이상 등)

### 2-3. 신청 이력 조회

- [ ] **개인 신청 이력**
  - [ ] 내 근태 신청 목록 조회 (전체/필터링)
  - [ ] 신청 상태별 필터링
  - [ ] 기간별 검색
  - [ ] 근태 유형별 필터링
  - [ ] 신청 상세 정보 조회

- [ ] **신청 통계**
  - [ ] 월별/연도별 신청 건수
  - [ ] 근태 유형별 사용 현황
  - [ ] 잔여 연차 일수 표시

#### 완료 기준
- 직원이 모든 근태 유형을 신청할 수 있음
- 신청 시 모든 유효성 검증이 정상 작동
- 워케이션 신청 시 필요한 모든 정보를 입력할 수 있음
- 워케이션 중 체크인/체크아웃이 정상 작동함
- 일일 업무 보고를 제출하고 조회할 수 있음
- 관리자가 워케이션 중인 직원을 원격으로 모니터링할 수 있음
- 위치 정보가 정확히 기록되고 관리됨 (동의 시)
- 근무 시간이 자동으로 집계됨
- 신청 이력을 조회하고 통계를 확인할 수 있음

## 📌 7. 데이터베이스 설계

### 7-1. 핵심 테이블

- [ ] **근태 유형 테이블 (leave_types)**
  ```
  - leave_type_id (PK)
  - type_name (반차/월차/연차/휴가/워케이션)
  - type_code
  - max_days (최대 신청 일수)
  - min_advance_days (최소 사전 신청 일수)
  - approval_steps (승인 단계 수)
  - is_active (활성화 여부)
  - created_at, updated_at
  ```

- [ ] **근태 신청 테이블 (leave_requests)**
  ```
  - request_id (PK)
  - employee_id (FK)
  - leave_type_id (FK)
  - start_date
  - end_date
  - half_day_type (오전/오후, 반차인 경우)
  - reason (사유)
  - attachment_url (첨부 파일)
  - work_location (워케이션 장소)
  - work_purpose (워케이션 목적)
  - status (대기/승인/반려/취소)
  - current_approval_step
  - created_at, updated_at
  ```

- [ ] **근태 승인 테이블 (leave_approvals)**

  - [ ] 모든 권한

## 📌 18. 참고사항

### 기술 스택 추천
- **Backend**: Node.js / Python / Java
- **Frontend**: React / Vue.js / Angular
- **Database**: PostgreSQL / MySQL
- **Cloud**: AWS / Azure / GCP
- **인증**: JWT / OAuth 2.0

- 개인정보보호법 준수
- 다중 테넌트 지원 시 데이터 격리 필수
