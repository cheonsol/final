# 웹툰/웹소설 에이전시 건강관리 시스템 UI 기획서

> **목표**: 관리자와 작가 모두가 직관적으로 사용할 수 있는 건강관리 대시보드 및 인터페이스 설계

---

## 1. 전체 UI 아키텍처

### 사용자 역할별 인터페이스 분리
- **관리자 대시보드**: 전체 작가 건강 현황 모니터링 및 관리
- **작가 개인 페이지**: 개인 건강 데이터 입력 및 조회
- **공통 컴포넌트**: 설문, 알림, 차트 등 재사용 가능한 UI 요소

### 반응형 디자인 원칙
- **데스크톱 우선**: 관리자는 주로 PC에서 사용
- **모바일 친화적**: 작가들의 간편한 체크인을 위한 모바일 최적화
- **다크모드 지원**: 장시간 화면 작업하는 작가들을 위한 눈의 피로 감소

### 4.2 인터랙티브 차트 컴포넌트
- **시계열 차트**: 작업시간, 통증 지수, 컨디션 변화 추이
- **히트맵**: 주간/월간 작업 패턴 시각화
- **레이더 차트**: 종합 건강 지수 (작업강도, 스트레스, 신체 컨디션 등)
- **진행률 바**: 건강검진 수검률, 목표 달성률 등

## 6. 접근성 및 사용성 고려사항

### 6.1 시각적 접근성
- **고대비 모드**: 시력이 약한 사용자를 위한 고대비 테마
- **폰트 크기 조절**: 사용자 설정에 따른 텍스트 크기 변경
- **색상 의존도 최소화**: 색맹 사용자를 위한 아이콘/패턴 병행 사용

### 6.2 인지적 부담 최소화
- **단순한 선택지**: 5점 척도보다는 3-4점 척도 사용
- **시각적 피드백**: 입력 완료 시 명확한 성공 메시지
- **진행률 표시**: 설문이나 체크인 과정의 진행 상황 표시
- **자동 저장**: 설문 중간 이탈 시 자동 저장 후 이어서 작성 가능

### 6.3 습관 형성 지원
- **연속 체크인 배지**: 게이미피케이션 요소로 동기 부여
- **개인 기록 달성**: "이번 주 5일 연속 체크인 완료!" 등
- **부드러운 리마인더**: 강압적이지 않은 친근한 톤의 알림
- **스트레칭 실천 기록**: 스트레칭 완료 시 기록 및 배지 부여

## 7. 데이터 시각화 전략

### 7.1 관리자용 고급 차트
- **실시간 대시보드**: 현재 접속 중인 작가 수, 위험군 현황

- **히트맵**: 시간대별/요일별 건강 이슈 발생 패턴

- **간단한 트렌드**: 복잡한 차트보다는 직관적인 화살표와 색상
- **개인 목표 추적**: "이번 주 목표: 야간작업 2회 이하" 등

- **건강 예측 그래프**: 연속 출근 시 예상 건강 점수 변화 추이

### 8.1 기본 API 구조
```
Base URL: https://api.health-management.company.com/v1
Authentication: Bearer Token (JWT)
Content-Type: application/json
```

### 8.2 주요 API 엔드포인트

#### 인증 및 사용자 관리
```
POST /auth/login                    # 로그인
POST /auth/logout                   # 로그아웃
GET  /auth/me                       # 현재 사용자 정보
PUT  /auth/me                       # 사용자 정보 수정
```

#### 출퇴근 및 건강 체크
```
POST /attendance/check-in           # 출근 체크
POST /attendance/check-out          # 퇴근 체크
GET  /attendance/status             # 현재 출퇴근 상태
GET  /attendance/consecutive-days   # 연속 출근 일수 조회
POST /health/daily-check            # 일일 건강 상태 체크 (강화된 설문 포함)
GET  /health/daily-check/status     # 오늘 건강 체크 완료 여부
```

#### 건강 데이터 관리
```
GET  /health/dashboard              # 개인 건강 대시보드 데이터
GET  /health/reports/{period}       # 건강 리포트 (weekly/monthly)
POST /health/symptoms               # 증상 보고
GET  /health/recommendations        # 개인 맞춤 건강 제안
GET  /health/prediction             # 연속 출근 건강 수치 예측
GET  /health/optimal-pattern        # 개인별 최적 근무 패턴 제안
GET  /health/score                  # 현재 건강 점수 조회
GET  /health/trend                  # 건강 지표 추이 조회
```

#### 스트레칭 가이드 (신규)
```
GET  /stretching/recommendations    # 맞춤 스트레칭 추천
POST /stretching/complete           # 스트레칭 완료 기록
GET  /stretching/history            # 스트레칭 실천 기록
```

#### 설문 관리
```
GET  /surveys/weekly                # 주간 설문 조회
POST /surveys/weekly                # 주간 설문 제출
GET  /surveys/monthly               # 월간 설문 조회
POST /surveys/monthly               # 월간 설문 제출
GET  /surveys/status                # 설문 제출 현황 조회
GET  /reports/monthly-summary       # 월간 종합 건강 리포트 자동 생성
```

#### 관리자 전용
```
GET  /admin/dashboard               # 관리자 대시보드
GET  /admin/users                   # 전체 사용자 현황
GET  /admin/alerts                  # 위험군 알림 목록
GET  /admin/consecutive-alerts      # 연속 출근 경고 대상자 목록
POST /admin/interventions           # 관리자 개입 조치 기록
GET  /admin/reports/aggregate       # 전체 통계 리포트
GET  /admin/surveys/pending         # 설문 미제출자 목록
GET  /admin/health-trend            # 전사 건강 트렌드 분석
```

#### 기본 알림
```
GET  /notifications                 # 알림 목록
PUT  /notifications/{id}/read       # 알림 읽음 처리
POST /notifications/email-settings  # 이메일 알림 설정
POST /notifications/stretching      # 스트레칭 알림 설정
```

### 위험군 분류 기준

| 분류 | 조건 | 조치 |
| --- | --- | --- |
| 🔴 고위험군 | 건강점수 60점 미만 OR 연속출근 14일+ OR 통증 7점+ | 즉시 조치: 휴재 권고, 상담 예약, 장비 지급 |
| 🟡 주의군 | 건강점수 60~69점 OR 연속출근 7~13일 OR 피로도 7점+ | 모니터링 강화: 체크인 독려, 휴식 권고 |
| 🟢 정상군 | 건강점수 70점 이상 AND 연속출근 6일 이하 | 현재 패턴 유지, 정기 설문 참여 |

#### 위험군 분류 상세 근거
| 분류 기준 | 근거 |
| --- | --- |
| 건강점수 60점 미만 = 고위험 | 6개 지표 중 3개 이상이 경고 수준일 때 도달하는 점수. 산업안전보건법상 "건강장해 우려" 수준으로 즉시 개입 필요 |
| 연속출근 14일+ = 고위험 | 근로기준법 휴일 규정 2주 연속 위반. 산업안전보건연구원 연구상 재해 위험 2배 증가 구간 |
| 통증 7점+ = 고위험 | VAS 기준 중증 통증(75mm+). 의료적 개입 없이 작업 지속 시 만성화 위험 |
| 피로도 7점+ = 주의군 | FSS 기준 임상적 피로 수준. 휴식 없이 지속 시 고위험군 전환 가능성 높음 |

### 기준 설정의 한계 및 주의사항

| 한계점 | 설명 | 해결 방안 |
| --- | --- | --- |
| **개인차 존재** | 동일 점수라도 개인별 체감 및 영향이 다를 수 있음 | • 개인별 기준선(baseline) 설정: 최초 2주간 데이터로 개인 평균 산출 후 상대적 변화 추적|
| **자가 보고 의존** | 객관적 측정이 아닌 주관적 설문 기반으로 정확도 한계 | • 복수 지표 교차 검증: 단일 지표가 아닌 여러 지표의 조합으로 판단<br>• 이상치 탐지: 평소 패턴과 급격히 다른 응답 시 재확인 요청<br>• 웨어러블 연동 (선택): 스마트워치 수면 데이터 연동으로 객관적 보완<br>• 관리자 면담 병행: 고위험군 분류 시 1:1 면담으로 실제 상태 확인 |
| **업계 특수성** | 웹툰/웹소설 작가의 불규칙한 작업 패턴 반영 필요 | • 마감 주기 반영: 연재 주기(주간/격주/월간)에 따른 가중치 자동 조정<br>• 야간 작업자 모드: 주 작업 시간대가 야간인 경우 별도 기준 적용<br>• 프로젝트 상태 연동: 마감 D-3 이내 시 일시적 기준 완화 후 마감 후 회복 모니터링 |
| **법적 기준과의 차이** | 본 시스템의 기준은 법적 강제 기준이 아닌 내부 관리 기준 | • 법적 기준 별도 표시: 법적 의무 사항과 권장 사항 구분 표기<br>• 법령 업데이트 모니터링: 관련 법령 개정 시 시스템 기준 동기화<br>• 면책 조항 명시: 시스템 권고가 의료/법적 조언이 아님을 안내 |

#### 추가 개선 방안
- **분기별 기준 리뷰**: 축적된 데이터 분석 후 임계값 조정
- **A/B 테스트**: 새로운 기준 적용 전 일부 그룹 대상 테스트
- **피드백 수집**: 작가/관리자 의견 반영한 기준 개선
- **버전 관리**: 기준 변경 이력 기록 및 이전 버전과 비교 분석
