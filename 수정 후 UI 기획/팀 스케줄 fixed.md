📘 요구사항 정의서  
팀별 캘린더 & 휴재 일정 대시보드  
웹 작가 에이전시 HR 시스템

---

## 1. 문서 개요

### 1.1 목적
본 요구사항 정의서는  
웹 작가 에이전시에서 발생하는 연재·휴재 정보를 기반으로,  
팀·작품 단위의 운영 현황을 한눈에 파악할 수 있는  
캘린더 중심 운영 대시보드를 구현하기 위한 요구사항을 정의한다.

본 시스템은 개인 일정 데이터를 포함하되,  
개인 일정 관리가 아닌 에이전시 운영 판단을 지원하는 것을 목적으로 한다.

---

### 1.2 적용 범위
- 팀/작품 단위 캘린더
- 휴재 중심 일정 관리
- 장기·시즌 휴재 관리
- 운영자 관점 요약 패널
- 권한별 캘린더 접근 제어
- UI 요구사항 정의 (와이어 수준)

---

### 1.3 제외 범위
- 개인 건강 설문 로직 상세
- 체크인/퇴근 처리 로직
- Todo 및 작업 진행률 계산 방식
- 워케이션 내부 운영 플로우

---

## 2. 사용자 유형 및 권한

### 2.1 사용자 유형
- 작가: 본인 작품 일정 조회
- 편집자: 담당 작품/팀 일정 조회
- 팀장: 소속 팀 전체 일정 조회
- 관리자: 전체 에이전시 일정 조회

---

### 2.2 권한별 접근 범위 요구사항
- 사용자는 본인 권한 범위를 초과하는 팀/작품을 조회할 수 없다.
- 권한에 따라 캘린더 필터가 자동 적용된다.
- 권한 없는 작품은 UI 목록에 노출되지 않는다.

---

## 3. 시스템 핵심 개념 정의

### 3.1 연재 일정
- 작품 등록 시 설정되는 기본 공개 흐름
- 월 단위로 자동 생성
- 삭제되지 않으며 항상 기준선 역할 수행

### 3.2 휴재 일정
- 연재 일정 변경을 표현하는 상태 이벤트
- 연재 일정 자체를 삭제하지 않음
- 모든 일정 변경은 휴재 이벤트로 기록

### 3.3 휴재 유형
- 정기 휴재
- 건강 휴재
- 긴급 휴재
- 시즌 휴재
- 일상 휴식 (연재 영향 없음)

---

## 4. 기능 요구사항

### 4.1 캘린더 조회
- 월 단위 캘린더 조회 가능
- 팀/에이전시 선택에 따른 범위 변경 가능
- 캘린더 기준 축은 팀·작품 단위

---

### 4.2 연재 일정 표시
- 작품별 정규 연재 요일 반복 표시
- 연재 일정은 항상 기본 흐름으로 유지
- 작품명 및 연재 플랫폼 정보 함께 표시

---

### 4.3 휴재 일정 표시 (핵심)
- 승인 완료된 휴재 일정만 표시
- 연재 일정 위에 덮어쓰기 방식
- 휴재는 의도된 운영 중단 상태로 인식되어야 함

#### 승인 상태 표시
- 승인 대기: 반투명 + 상태 태그
- 승인 완료: 불투명
- 반려: 빨간색 테두리

#### 휴재 기간 강조
- 1~3일: 연한 회색
- 4~14일: 회색
- 15일 이상: 진한 회색 + 경고 아이콘
- “N일째 휴재 중” 텍스트 표시

---

### 4.4 경고 및 강조
- 휴재 기간과 마감일이 겹치는 경우 경고 표시
- 휴재 집중 기간 발생 시 배경 강조 및 라벨 표시

---

### 4.5 요약 패널
- 팀별 휴재 건수
- 휴재 유형 분포
- 장기 휴재 작품 목록
- 휴재 일수 및 복귀 예정일
- 휴재 관리 상태 (정상/주의/위험)
- 전월 대비 휴재 증감률

캘린더와 동일 데이터 기준으로 집계되어야 한다.

---

## 5. 비즈니스 로직 요구사항

### 5.1 연재 일정 생성
- 작품 등록 시 연재 요일 설정
- 월 단위 연재 일정 자동 생성

### 5.2 휴재 반영 흐름
1. 작가 휴재 신청 (작품, 기간, 유형)
2. 승인 완료
3. 해당 기간 연재 일정 상태 CANCELED 처리
4. 휴재 이벤트 생성
5. 작품 단위 휴재 이력 누적

---

## 6. UI/UX 요구사항

### 6.1 기본 원칙
- 클릭 없이 상태 파악 가능
- 색상·패턴·투명도로 상태 구분
- 운영 리스크 즉시 인지 가능

### 6.2 색상 규칙
- 연재: 초록
- 휴재: 회색/패턴
- 승인 대기: 반투명
- 승인 완료: 불투명
- 반려: 빨간 테두리
- 장기 휴재: 진한 회색 + 경고 아이콘

---

## 7. 기술 및 시스템 요구사항

### 7.1 프론트엔드
- React 기반
- FullCalendar 활용한 팀·작품 캘린더 구성

### 7.2 백엔드
- 연재/휴재 이벤트 사전 생성
- 기간 기반 조회 최적화
- 휴재 승인 트랜잭션 처리

---

## 8. 비기능 요구사항
- 월 단위 캘린더 조회 1초 이내 응답
- 캘린더와 요약 패널 데이터 정합성 보장
- 권한 범위 초과 데이터 노출 방지

---

## 9. 요구사항 요약 정의

본 시스템은  
작가 개인의 연재·휴재 데이터를 기반으로,  
팀·작품 단위의 운영 현황을 캘린더와 요약 패널로 제공하는  
에이전시 전용 운영 대시보드이다.
